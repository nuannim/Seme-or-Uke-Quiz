# การทดสอบการปิด hover สำหรับโทรศัพท์สมาร์ทโฟน

ระบบต้องการทดสอบว่าเมื่อใช้หน้าจอสัมผัส (Touchscreen / Smartphone) ปุ่มและตัวเลือกต่าง ๆ จะไม่มีการติดค้างสถานะ hover เมื่อกดใช้งาน แต่ในทางกลับกัน บนคอมพิวเตอร์ที่ใช้เมาส์ชี้ (Mouse Pointer) จะต้องยังมีเอฟเฟกต์ hover แสดงขึ้นมาตามปกติ

---

## 1. การทดสอบการแสดงผล hover บนคอมพิวเตอร์ (Desktop Browser with Mouse)

- [ ] เปิดเว็บแอปพลิเคชันบนเบราว์เซอร์คอมพิวเตอร์ที่ใช้งานเมาส์
- [ ] ในหน้าเริ่มต้น (Welcome Screen):
  - [ ] เลื่อนเมาส์ไปชี้ที่ Brand Header Bubbles (วงกลมคำว่า "เมะ" และ "เคะ") แล้วตรวจสอบว่ามีแอนิเมชันแยกออกจากกันเล็กน้อย
  - [ ] เลื่อนเมาส์ไปชี้ที่ปุ่ม "เริ่มทำแบบทดสอบเลย!" แล้วตรวจสอบว่าปุ่มมีการยกตัวขึ้นเล็กน้อยและมีเงาขยายขึ้น
- [ ] กดเริ่มเล่นแบบทดสอบเพื่อเข้าสู่หน้าคำถาม:
  - [ ] เลื่อนเมาส์ไปชี้ที่ตัวเลือกตอบ (Choice Item) ต่าง ๆ แล้วตรวจสอบว่าตัวเลือกที่ชี้มีการเปลี่ยนสีพื้นหลังเป็นสีอ่อน ขอบสีเข้มขึ้น มีเงา และยกตัวขึ้นเล็กน้อย
  - [ ] ตัวเลขวงกลมหน้าตัวเลือก (Choice Badge) เปลี่ยนสีเป็นสีน้ำเงินเข้มตัวอักษรขาวเมื่อเอาเมาส์ชี้ตัวเลือกนั้น
- [ ] ทำแบบทดสอบให้เสร็จเพื่อไปที่หน้าผลลัพธ์:
  - [ ] เลื่อนเมาส์ไปชี้ที่ปุ่มการกระทำต่าง ๆ (เช่น บันทึกรูปภาพ, ทำใหม่อีกครั้ง) และปุ่มควบคุมเพลง (Mini-controls) แล้วตรวจสอบว่ามีเอฟเฟกต์ hover ตอบสนองตามปกติ

---

## 2. การทดสอบการไม่มี hover บนโทรศัพท์มือถือ (Smartphone/Touch Device or Mobile Emulation)

- [ ] เปิดเว็บแอปพลิเคชันบนสมาร์ทโฟนจริง หรือใช้ Mobile Emulation (เช่น Chrome DevTools Device Toolbar)
- [ ] ในหน้าเริ่มต้น (Welcome Screen):
  - [ ] ลองใช้นิ้วสัมผัสที่ปุ่ม หรือส่วนต่าง ๆ เพื่อดูว่าไม่มีเอฟเฟกต์ hover ค้างหลังจากสัมผัสเสร็จ
- [ ] กดเริ่มเล่นแบบทดสอบเพื่อเข้าสู่หน้าคำถาม:
  - [ ] ใช้นิ้วกดเลือกคำตอบในข้อที่ 1
  - [ ] [ตรวจสอบ] เมื่อเปลี่ยนเป็นข้อที่ 2 แล้ว ตัวเลือกคำตอบในตำแหน่งเดียวกันจะต้องมีสีพื้นหลังเป็นสีขาวปกติ (ไม่ติดค้างเป็นสีเทาหรือขอบสีเข้มที่เป็น hover state ของข้อก่อนหน้า)
  - [ ] [ตรวจสอบ] ทุกครั้งที่ตอบคำตอบเสร็จและคำถามข้อถัดไปขึ้นมา จะต้องไม่มีตัวเลือกใด ๆ แสดงผลเป็นสถานะ hover ค้างก่อนจะถูกกดสัมผัสจริง
- [ ] ทำแบบทดสอบจนเสร็จ:
  - [ ] ลองใช้นิ้วสัมผัสปุ่มและหน้าจอผลลัพธ์ แล้วตรวจสอบว่าไม่มีปุ่มใดที่แสดงสถานะ hover ค้างหลังจากการสัมผัสผ่านไปแล้ว
